# Maquettes

Avant de commencer le développement d'une application, il est recommandé de passer par plusieurs étapes de conception visuelle. Ces étapes permettent de valider les besoins, améliorer l'expérience utilisateur et réduire les erreurs de développement.

## Pourquoi réaliser des maquettes ?

La création de maquettes présente plusieurs avantages :

- **Visualiser le produit avant le développement**.
- **Valider les besoins fonctionnels** avec les parties prenantes.
- **Identifier les problèmes d'ergonomie** dès les premières phases du projet.
- **Faciliter les échanges** entre les équipes métier, UX/UI et développement.
- **Réduire les coûts de modification** en détectant les erreurs avant l'implémentation.
- **Servir de référence** tout au long du développement.

---

## Wireframe

Le wireframe représente la structure générale d'une interface sans se concentrer sur l'aspect graphique.

### Objectifs

- Définir l'organisation des éléments.
- Structurer les parcours utilisateurs.
- Valider l'emplacement des fonctionnalités.
- Se concentrer sur l'expérience utilisateur avant le design.

## Mockup

Le mockup est une version plus avancée du wireframe intégrant l'identité visuelle du projet.

### Objectifs

- Visualiser le rendu final de l'application.
- Définir les couleurs, typographies et composants graphiques.
- Valider la cohérence de la charte graphique.
- Faciliter les retours des utilisateurs et des clients.

![screen.png](/img/maquettes/screen.png)

---

## Prototype

Le prototype est une version interactive permettant de simuler le comportement de l'application avant son développement.

### Objectifs

- Tester les parcours utilisateurs.
- Vérifier la navigation entre les différentes pages.
- Détecter les problèmes d'ergonomie.
- Obtenir des retours utilisateurs avant le développement.

## Maquette complète

Une maquette complète regroupe l'ensemble des écrans et parcours de l'application. Elle constitue la référence principale pour les équipes de développement.
