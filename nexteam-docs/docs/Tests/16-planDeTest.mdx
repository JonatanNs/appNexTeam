# Plan de tests

L’objectif de cette phase de test est de vérifier le bon fonctionnement de l’ensemble des fonctionnalités de l’application ConnecTeam, de valider les règles métier définies dans les user stories et de s’assurer que le système gère correctement les cas normaux ainsi que les cas d’erreurs.

---

# 1. Authentification

| ID | Cas de test | Entrée | Résultat attendu |
| --- | --- | --- | --- |
| T1 | Connexion valide | Email + mot de passe correct | Accès à la plateforme + token JWT |
| T2 | Connexion invalide | Mauvais mot de passe | Message d’erreur |
| T3 | Email inexistant | Email non enregistré | Refus d’accès |

# 8. Cas d’erreurs globaux

- Email déjà utilisé lors création user
- Mot de passe incorrect
- Accès refusé selon rôle
- Données manquantes dans requêtes
- Fichier trop volumineux ou invalide
