Plan de tests

L’objectif des tests est de valider :

  • le bon fonctionnement des règles métier
  • la robustesse du système face aux cas nominaux et cas d’erreurs
  • l’intégration correcte entre les couches (API → Service → Repository → DB)

Les tests sont divisés en deux niveaux :

  • Tests unitaires : validation isolée de la logique métier (Mockito, sans Spring context complet)
  • Tests d’intégration : validation du comportement global (API REST + Spring Boot + DB)

1. Tests unitaires (Service Layer)​

Objectif : tester la logique métier isolée avec dépendances mockées.
